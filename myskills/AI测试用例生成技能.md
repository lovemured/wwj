# AI测试用例生成技能

> **版本**: v1.1 | **适用AI**: ChatGPT / Claude / 通义千问 / 文心一言等主流大模型

---

## ⚡ 3分钟快速上手

```
┌─────────────────────────────────────────────────────────┐
│  步骤1: 复制需求 → 步骤2: 粘贴Prompt → 步骤3: 获取用例  │
│                    ↓                                    │
│              步骤4: 人工审核 → 步骤5: 导出执行            │
└─────────────────────────────────────────────────────────┘
```

**极简流程**：将需求文档粘贴到下方Prompt模板中，发送给AI即可获得测试用例。

## 📖 技能概述

本技能用于根据需求文档自动生成结构化测试用例，支持多种测试场景覆盖，输出标准Excel格式文件。

### 适用场景

| 场景 | 说明 | 建议Prompt版本 |
|-----|------|---------------|
| 新功能测试 | 需求评审后快速生成用例 | 基础版/进阶版 |
| 接口测试 | RESTful/API用例编写 | 进阶版（接口专项） |
| 回归测试 | 从大量用例中筛选关键用例 | 专项补充指令 |
| 安全测试 | 认证鉴权、渗透测试用例 | 专家版+安全专项 |
| 性能测试 | 并发、压力、负载测试 | 专家版+性能专项 |
| 移动端测试 | iOS/Android小程序测试 | 进阶版+兼容专项 |

### 预期产出

- ✅ 结构化测试用例（Markdown/Excel格式）
- ✅ 需求不明确点清单
- ✅ 测试数据建议
- ✅ 人工确认事项列表
- ✅ 覆盖率分析报告（专家版）

```
【需求文档】

建议预处理：

✓ 说明接口参数（如适用）
✓ 提供参考数据范围

根据需求复杂度选择合适的Prompt模板：

| 需求规模 | 建议版本 | 预估用例数 |

| 简单功能（<10个功能点） | 基础版 | 20-50条 |
| 中等需求（10-30个功能点） | 进阶版 | 50-150条 |
| 复杂系统（>30个功能点） | 专家版 | 150-500条 |

### 第四步：质量检查与导出

## 🧪 测试数据生成指南

### 自动生成测试数据的方法

在Prompt中追加以下指令，让AI生成测试数据：

```
请为每条测试用例补充：
1. 具体的测试数据值
2. 数据类型和格式要求
3. 边界值测试的具体数值
4. 特殊测试数据（如超长字符串、SQL注入字符）

示例格式：
| 用例ID | 测试数据 | 数据类型 | 预期行为 |
|-------|---------|---------|---------|
| LOGIN-001 | user123 | 正常6位 | 正常登录 |
| LOGIN-002 | abcde | 5位边界 | 提示长度不足 |
```

### 常见测试数据类型

| 类型 | 示例数据 |
|-----|---------|
| 正常值 | `admin123`、`张三`、`13800138000` |
| 边界值 | 空字符串、超长字符（1000+）、特殊字符 |
| 异常值 | SQL注入：`' OR '1'='1`、XSS：`<script>alert(1)</script>` |
| 格式错误 | 邮箱格式错误、手机号少一位 |
| 边界数值 | Integer.MAX_VALUE、负数、零 |

## ⚠️ 注意事项

### AI生成局限性

| 风险 | 说明 | 应对措施 |
|-----|------|---------|
| 业务理解偏差 | AI可能误解业务规则 | 人工审核关键业务规则 |
| 幻觉风险 | AI可能编造不存在的内容 | 使用❓标记，核实每个疑点 |
| 上下文限制 | 超长需求需分段处理 | 按模块拆分，分批生成 |
| 领域知识 | 特定行业规则需额外说明 | 提供行业背景和术语解释 |

### 最佳实践

1. **需求预处理** - 提前标注关键信息，减少AI理解偏差
2. **分批生成** - 大型需求按模块分批生成，保证质量
3. **交叉验证** - 使用多个AI工具对比，提高准确性
4. **人工审核** - 所有AI生成内容必须经过人工审核
5. **持续迭代** - 积累高质量用例模板，提升生成效率

## ❓ 常见问题

**Q: 生成的用例太简单怎么办？**
A: 追加指令："请深入分析业务逻辑，补充复杂场景和组合场景的测试用例"

**Q: 用例覆盖不全怎么办？**
A: 使用"测试场景覆盖清单"检查，追加缺失场景的生成指令

**Q: 如何处理超长需求文档？**
A: 按模块拆分，分批生成，最后合并

**Q: 如何保证用例质量？**
A: 使用"AI生成质量评估表"逐项评估，重点关注AI幻觉

**Q: 多个AI工具生成结果不同怎么办？**
A: 对比各版本，取最优用例合并，形成最终版本

**Q: 用例数量过多如何管理？**
A: 使用优先级筛选，先执行P0/P1用例
