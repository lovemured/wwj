# 接口测试覆盖清单（评审/自检用）

> 用法：对每个接口或每组接口（例如“订单域”）评审时逐项勾选；不适用项写清原因。  
> 建议：P0 接口要求“基础层 + 业务层 + 异常容错层”全覆盖，安全/性能/稳定按风险选配。

---

## A. 基础功能与契约
- [ ] 接口路径/方法/Content-Type 与文档一致
- [ ] 正向用例（典型输入）成功
- [ ] 返回状态码与业务码符合约定
- [ ] 响应结构字段存在性/类型/可空性校验
- [ ] 默认值与缺省行为（未传字段时）明确且被验证
- [ ] i18n/错误信息文案（如要求）一致

---

## B. 参数校验（VALID）
- [ ] 必填字段缺失
- [ ] 空值：空字符串、`null`、空数组/空对象（按字段类型）
- [ ] 类型错误：string/number/boolean/object/array
- [ ] 长度/范围边界：min-1、min、max、max+1
- [ ] 格式校验：日期/邮箱/手机号/UUID/正则
- [ ] 枚举：合法、非法、大小写（如不区分）
- [ ] 组合约束：字段间依赖（A 有值则 B 必填）
- [ ] 列表/数组：空、超长、重复、顺序敏感/不敏感

---

## C. 认证与权限（SEC）
- [ ] 未登录/无 Token
- [ ] Token 过期
- [ ] Token 非法/篡改
- [ ] 权限不足（角色/资源权限）返回 403（或约定业务码）
- [ ] 字段级权限/脱敏（不同角色返回字段差异）

### C1. 越权
- [ ] 水平越权：A 访问/修改/删除 B 的资源
- [ ] 垂直越权：低权限调用高权限接口

---

## D. 业务规则与一致性（FUNC）
- [ ] 关键业务规则（限额/库存/状态/风控）覆盖
- [ ] 状态流转：合法迁移、非法迁移
- [ ] 并发冲突：并发创建/并发更新/重复提交
- [ ] 数据一致性：DB、缓存、消息（如适用）均符合预期

---

## E. 异常与容错（VALID/STAB）
- [ ] 下游失败时：错误码/错误信息/可恢复性符合约定
- [ ] 下游超时时：超时策略（快速失败/降级）符合约定
- [ ] 重试策略：客户端/网关/服务端重试是否造成重复副作用
- [ ] 幂等（如声明支持）：幂等键重复提交结果一致

---

## F. 稳定性与兼容性（STAB）
- [ ] 超时设置符合 SLA（不会无限等待）
- [ ] 限流：触发限流返回码、提示、`Retry-After`（如有）
- [ ] 熔断/降级：触发与恢复行为可验证（如有）
- [ ] 版本兼容：新增字段旧客户端可忽略；废弃字段有过渡期
- [ ] 枚举新增：客户端对未知枚举有兜底

---

## G. 性能与压测（PERF）
- [ ] 明确目标指标：P95/P99、QPS、错误率
- [ ] 基线压测（稳定负载）
- [ ] 阶梯压测（逐步升压）
- [ ] 峰值冲击（短时突增）
- [ ] 大包体/大列表/大字段场景（如适用）
- [ ] 长稳压测（30min/2h）观察抖动与资源泄露（如必要）
- [ ] 观测项：CPU、内存、GC、线程池、DB 连接池（如可观测）

---

## H. 可观测与审计（建议）
- [ ] 关键请求具备 requestId/traceId 便于定位
- [ ] 错误日志不泄露敏感信息（Token/密码/身份证）
- [ ] 审计日志记录关键行为（如有合规要求）

---

## I. 自动化与可维护性
- [ ] 用例可自动化（或明确不可自动化原因）
- [ ] 测试数据可复现（统一造数/夹具/脚本）
- [ ] 用例断言稳定（避免强依赖时间/随机/外部系统）
- [ ] CI 触发策略明确（冒烟/回归/发布前）
